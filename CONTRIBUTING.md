# Contributing to Awesome Agent Bounties

Thanks for your interest in contributing! This list thrives on community contributions.

## Quick Start

1. Fork this repository
2. Add your entry to the appropriate category in `README.md`
3. Open a Pull Request

## Adding a New Bounty Program

### Entry Format

Add your project to the relevant category table using this format:

```markdown
| **[Project Name](https://github.com/link))** | TOKEN | $MIN-$MAX | Brief description | [Bounties](link) |
```

### Required Information

- **Project name**: Link to main repo or website
- **Token/Currency**: What you pay in (USDC, ETH, RTC, etc.)
- **Bounty range**: Min-max typical payout
- **Focus**: 2-5 words describing the work
- **Link**: Direct link to bounty page/issues

### Quality Criteria

Your project should:

- ✅ Pay contributors (money or tokens with real value)
- ✅ Have a public, accessible bounty tracker
- ✅ Accept submissions from anyone (no invite-only programs)
- ✅ Be actively maintained

**Not accepted:**
- ❌ Closed/invite-only programs
- ❌ "Exposure" or "portfolio building" only
- ❌ Dead projects with no activity

## Improving Existing Entries

You can also:

- Update outdated bounty ranges
- Fix broken links
- Improve descriptions
- Add missing categories

## Style Guide

- **Alphabetical order** within categories
- **Concise descriptions** (2-5 words)
- **Working links** - verify before submitting
- **No promotional language** - stick to facts

## Pull Request Checklist

- [ ] Entry added to correct category
- [ ] Alphabetically sorted within category
- [ ] All links verified and working
- [ ] Bounty range is accurate
- [ ] Description is concise (2-5 words)

## Questions?

Open an issue for discussion before making major changes.
